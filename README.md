# Soul Online Alpha 0.88 덮어쓰기 패치

Soul Online 0.88은 0.87의 기술 구조 분리 기반 위에서 가방/상점 UX와 기술 점검을 다듬는 안정화 패치입니다. 0.87 위에 그대로 덮어쓰는 소스 패치로 사용할 수 있습니다.

## 적용 순서

1. GitHub Desktop에서 현재 상태를 먼저 커밋합니다.
2. 이 ZIP을 프로젝트 루트에 압축 해제해 덮어씁니다.
3. `npm install`을 실행합니다.
4. `npm run build`로 빌드 확인을 합니다.
5. Firebase Hosting 사용 시 `firebase deploy --only hosting`으로 배포합니다.

## 0.88 핵심

- 버전을 `0.88.0`으로 갱신했습니다.
- PWA 캐시를 `soul-online-alpha-v0-88`로 갱신했습니다.
- `src/ui/townPanelRenderer088.ts`를 추가해 가방/상점 필터와 구매 확인 UI 렌더러를 분리했습니다.
- `src/styles/alpha088.css`를 추가해 0.88 전용 safe-frame, 필터 탭, 구매 확인 패널 스타일을 분리했습니다.
- 가방에 필터 상태를 추가했습니다: 전체/장비/소모품/재료/스킬서.
- 상점에 필터 상태를 추가했습니다: 전체/소모품/스킬북/장비/재료.
- 상점 구매는 즉시 구매가 아니라 구매 확인 패널을 거치도록 변경했습니다.
- 상점 재고 연결 문제를 수정했습니다. 0.83 상점에 표시되던 `lumina-seal`, `radiant-ore`가 실제 구매 가격 테이블에도 연결됩니다.
- System Doctor/Tech Health에 렌더 예산(DOM/이미지/표시 패널) 점검을 추가했습니다.
- UI 안전 영역 검사에 0.88 가방 필터/상점 필터/구매 확인 패널을 포함했습니다.

## 계속 남은 과제

- `main.ts`는 아직 큽니다. 다음 단계에서는 가방/상점/스킬 렌더러를 더 많이 파일 단위로 분리해야 합니다.
- `styles.css`는 아직 거대합니다. 0.87/0.88처럼 신규 스타일은 별도 CSS로 분리해야 합니다.
- 대형 스프라이트 시트의 모바일 첫 로딩 최적화가 필요합니다.
