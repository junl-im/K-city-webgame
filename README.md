# 소울 온라인 Alpha 0.6 Daily Quest

모바일 웹 2.5D MMORPG 프로토타입입니다. 0.6부터 게임명을 **소울 온라인**으로 변경했고, 마을 허브에 일일 의뢰 시스템을 추가했습니다.

## 0.6에서 바뀐 점

- 게임명 변경
  - `SOL Online` → `소울 온라인`
  - HTML 타이틀, PWA manifest, 앱 캐시명, 패키지명을 갱신했습니다.

- 일일 의뢰 시스템 추가
  - 마을 컨텐츠에 `일일 의뢰` 패널 추가
  - 슬라임/늑대/고블린 처치, 레벨 달성형 목표 제공
  - 오늘 처치 수를 별도로 기록
  - 보상 수령 시 골드, 소울젬, 재료 지급
  - 날짜가 바뀌면 진행도와 수령 상태 초기화

- 저장 데이터 버전 갱신
  - `SAVE_VERSION = 3`
  - 기존 0.5 캐릭터는 자동 마이그레이션됩니다.
  - 기존 로컬 저장 키는 유지해서 이전 캐릭터가 사라지지 않게 했습니다.

## 기존 주요 기능

- 로그인: 게스트, Google, 로컬 저장
- 단일 서버: `곰같은여우 서버`
- 캐릭터 슬롯 최대 4개
- 캐릭터 생성, 선택, 삭제
- 마을 허브 시작
- 사냥터/레이드터 입장 및 마을 복귀
- 카드 장착/해제
- 장비 장착/해제
- 상점 구매
- 월드 보스 입장 패널
- 전투 로그 접기/열기
- Fullscreen API + PWA manifest
- Firebase Auth/Firestore 클라우드 저장 옵션

## 실행

```bash
npm install
npm run dev
```

## 빌드

```bash
npm run build
```

## Firebase 메모

클라우드 저장은 `/users/{uid}`와 `/rankings/{uid}`에 씁니다. Firestore Rules에 두 경로가 허용되어 있어야 합니다.

클라우드 저장이 실패해도 로컬 저장은 계속 유지됩니다. 계정/저장 패널에서 최근 클라우드 오류를 확인할 수 있습니다.
