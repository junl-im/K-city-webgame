# Soul Online Alpha 1.20 Recovery Kernel Boot Fix

## 원인 정리

첫 실행 문제가 생긴 핵심 원인은 한 가지 파일만의 오류라기보다, 여러 세대의 보정 레이어가 누적되면서 부팅 경로가 복잡해진 것이다.

- 1.14: 대형 CSS를 지연 로드로 분리했다.
- 1.15: 에셋 전달 경로를 바꿨다.
- 1.16: HUD/필드 배치를 강하게 보정했다.
- 1.17: 단일 화질 모드와 실행 크기 고정을 넣었다.
- 1.18: 장면 안정화 가드를 추가했다.
- 1.19: 긴급 부팅 CSS를 넣었지만, main.ts의 정적 import와 구형 보정 스택은 여전히 많이 남아 있었다.

이 조합 때문에 모바일에서 첫 화면이 뜬 뒤 START가 늦거나, 눌러도 로그인 버튼 바인딩이 늦거나, 장면 가드가 다시 타이틀/로그인 표시를 덮어쓸 수 있었다.

## 1.20 조치

- `src/ui/recoveryKernel120.ts` 추가
- `src/styles/alpha120.css` 추가
- `index.html`에 프리부트 START 핸들러 추가
- `main.ts`에서 START 클릭 시 페이드/구형 sync 없이 로그인 화면 즉시 표시
- 로그인/조작 이벤트 바인딩을 Firebase/Auth 초기화보다 먼저 실행
- Firebase/Auth 초기화는 백그라운드 처리
- 0.93~1.18 구형 비주얼 보정 스택은 기본 비활성화
- 실행 당시 화면 크기만 1.20 커널에서 고정

## 검증 기준

- 사이트 첫 진입 시 타이틀이 보여야 한다.
- JS 로딩이 느려도 START 터치가 로그인 화면을 열어야 한다.
- Firebase가 느려도 로컬 저장/로컬 접속 흐름이 막히면 안 된다.
- 장면 전환 중 타이틀/로그인/마을/필드가 동시에 표시되면 안 된다.
